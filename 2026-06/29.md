# Day 25 - SQL Constraints

## What I learned

SQL constraints are rules applied to table columns to ensure the accuracy and integrity of data.

### PRIMARY KEY

```sql
CREATE TABLE students (
    student_id INT PRIMARY KEY,
    name VARCHAR(50),
    marks INT
);
```

### NOT NULL

```sql
CREATE TABLE employees (
    employee_id INT,
    name VARCHAR(50) NOT NULL
);
```

### UNIQUE

```sql
CREATE TABLE users (
    user_id INT PRIMARY KEY,
    email VARCHAR(100) UNIQUE
);
```

### FOREIGN KEY

```sql
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

## Why Constraints are Useful

- Maintain data accuracy
- Prevent invalid data entry
- Establish relationships between tables

## Key Takeaway

Constraints improve data quality and ensure database consistency.

## Practice Example

```sql
CREATE TABLE products (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100) NOT NULL,
    price DECIMAL(10,2)
);
```
