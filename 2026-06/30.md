# Day 26 - SQL Views

## What I learned

A SQL View is a virtual table created from the result of a SQL query. It does not store data itself but displays data from one or more tables.

### Creating a View

```sql
CREATE VIEW High_Salary_Employees AS
SELECT employee_id, name, salary
FROM employees
WHERE salary > 50000;
```

### Viewing Data

```sql
SELECT * FROM High_Salary_Employees;
```

### Deleting a View

```sql
DROP VIEW High_Salary_Employees;
```

## Why Views are Useful

- Simplify complex queries
- Improve data security
- Reuse frequently used queries
- Make reporting easier

## Key Takeaway

Views help present specific data without changing the original tables.

## Practice Example

```sql
CREATE VIEW Top_Students AS
SELECT student_id, name, marks
FROM students
WHERE marks >= 90;

SELECT * FROM Top_Students;
```
